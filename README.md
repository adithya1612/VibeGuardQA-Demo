# VibeGuard QA™ — Public Demo Repository

Welcome to the public demo repository for **VibeGuard QA™** — a zero-infrastructure, local quality assurance and compliance framework for AI-generated codebases.

This repository demonstrates the VibeGuard scan-and-repair workflow on a typical Node.js/Express application generated by an AI assistant.

---

## Repository Structure

- `broken-app/`: A raw, AI-generated Express application with typical security, operation, and configuration vulnerabilities.
- `repaired-app/`: The secure, robust version of the application after running VibeGuard audits and applying the AI repair prompts.
- `scanner-report.md`: A sample markdown compliance report generated by the VibeGuard CLI scanner.

---

## The Workflow Demo

Follow these steps to experience the complete VibeGuard cycle:

### 1. Run the Scanner against the Broken App

If you have downloaded the **VibeGuard QA™ CLI Scorer**, navigate to the project directory and run the scanner script:

```bash
# Run scanner against the broken application
python core/vibeguard-score.py demo-repo/broken-app/
```

#### What VibeGuard detects:
1. **VG-SEC-001 (Hardcoded Secrets):** Plain-text Stripe API keys and database password strings inside the server script.
2. **VG-SEC-002 (SQL Injection):** JavaScript backtick query interpolation that allows malicious SQL parameters.
3. **VG-SEC-003 (Docker Root Privileges):** A Dockerfile omitting the `USER` instruction, running the runtime container as root.
4. **VG-OPS-001 (Missing Health Endpoint):** The web server lacks a `/health` status endpoint for load balancer probes.
5. **VG-OPS-003 (Missing Correlation IDs):** Web server logs are not trace-linked via request-unique headers.
6. **VG-OPS-004 (Connection Leaks):** MySQL client connections are initialized per request but never end, leaking database handles under load.

**Baseline Score:** **50% (FAIL)** (Compliance Gate: >=80%)

---

### 2. Inject the VibeGuard QA™ System Prompt

Copy the rules from `vibeguard-standalone.md` (or let your IDE pick them up automatically via `.cursorrules` / `CLAUDE.md`) to instruct your AI assistant (Cursor, Claude Code, or Copilot).

---

### 3. Apply the Repair Prompt

Provide your AI assistant with the scanner outputs or run the following instruction:

> *"Harden this application according to the VibeGuard standards. Parameterize queries, migrate keys to environment variables, close connections properly, add JSON structured logging with correlation IDs, include a database health-check probe, and configure the Dockerfile to run as the non-root user `node`."*

The AI assistant will automatically generate the clean version shown in `repaired-app/`.

---

### 4. Re-run the Scanner to Verify Compliance

Run the local scanner again to verify that all deductions have been resolved:

```bash
# Run scanner against the repaired application
python core/vibeguard-score.py demo-repo/repaired-app/
```

**Final Score:** **100% (PASS)** (Compliance Gate: >=80%)
